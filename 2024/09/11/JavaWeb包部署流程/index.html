JavaWeb包部署流程

2024/9/11

IDEA 安装

进入IDEA官网下载 IntelliJ IDEA – the Leading Java and Kotlin IDE (jetbrains.com)

学信网提供学籍档案 中国高等教育学生信息网（学信网） (chsi.com.cn)

Tomcat 安装

下载

  • 进入Tomcat官网 Apache Tomcat® - Welcome!

  • 点击左侧**Which version?**查看对应版本信息

Final Released Version 为目前放出版本

Supported java Version 为支持java版本，向更高版本兼容

  • 了解所选择版本后，Tomcat8.5维护时间较新推荐使用，此处笔者选择Tomcat10，点击左侧对应版本Tomcat，进入下载界面

  • 选择对应版本下载完成后，进行解压

配置

在此之前，请先确保正确配置了JAVA_HOME环境变量，如有出错会在下文集中处理

  • 打开 环境变量 配置界面，点击下方 系统变量-新建 ，因系统差异，请自行查询如何打开

  • 新建 CATALINA_HOME 系统变量，
    • 变量名：CATALINA_HOME
    • 变量值：D:\DevelopTools\apache-tomcat-10.1.28 此处为Tomcat根目录路径

  • 编辑 PATH 系统变量

    • 选择 PATH 变量，点击编辑

    • 点击新建，依次添加 %CATALINA_HOME%\lib %CATALINA_HOME%\bin %CATALINA_HOME%\lib\servlet-api.jar

测试

  • 进入Tomcat解压目录，进入bin文件夹，右键 在终端打开，运行 startup.bat，如果配置正常则出现以下信息，并额外出现一个窗口

  • 进入浏览器输入网址 localhost:8080 进入网页查看

  • 运行 shutdown.bat 来关闭Tomcat 不推荐直接关闭ps窗口来关闭Tomcat

报错

出现窗口但立刻消失

  • 确保 JAVA_HOME 系统变量已配置或配置正确

  • CATALINA_HOME中的路径不能以 \ 结尾。

  • 使用 PowerShell 等终端打开 bat文件 而不是直接双击运行

运行 start.bat 报错

  • 确保 JAVA_HOME 系统变量已配置或配置正确

Tomcat终端乱码

  • 在 .\apache-tomcat-XXXX\conf 下打开 logging.properties文件

  • 将java.util.logging.ConsoleHandler.encoding = UTF-8
    替换为java.util.logging.ConsoleHandler.encoding = GBK
